## 2. Natal Collection / Baseline Setup Flow - COMPLETE ✅

### Flow Structure (4 steps):

**Step 1: Welcome**
- "Welcome to DEFRAG"
- "A relational intelligence platform that helps you see interactions from more than one side"
- Clean, cinematic entry
- Premium on desktop and iPhone

**Step 2: Why We Ask**
- "Build Your Baseline"
- Explains: "DEFRAG uses your birth details to create a foundational relational profile"
- Four lenses explained simply:
  - **Astrology:** timing, pressure, active themes
  - **Human Design:** energy processing, decisions, stress
  - **Gene Keys:** recurring growth patterns, emotional themes
  - **Numerology:** cycles, emphasis, timing patterns
- Privacy note: "Birth details never shared; DEFRAG compares synthesized patterns, not raw data"
- Non-mystical, anti-stigma language throughout

**Step 3: Profile Entry**
- Birth date (required)
- Birth time (optional, with "use 12:00 PM if unknown" helper)
- Birth location (required, "City, State/Country" placeholder)
- Privacy explanation: "Used to generate baseline, never exposed to other users"
- Form validation: Continue button disabled until required fields complete

**Step 4: Completion**
- "Your Baseline Is Ready"
- Explains what happens next:
  1. Dashboard shows baseline and relational context
  2. Bring moments to workspace for clarity
  3. Add relationships to see how dynamics shift
- CTA: "Go to Dashboard" → routes to `/dashboard`

### Premium Quality:
- Ambient drifting backgrounds (20s animations)
- Smooth step transitions
- First-class mobile experience
- Clear, calm, human copy throughout

## 3. AI Output Structure - ALIGNED WITH DEFRAG REASONING ✅

### Visible Output Structure:
Every Defrag response follows this pattern:

1. **Short acknowledgment**
   - "They likely interpreted..."
   - System indicator: "Analyzing" with pulse animation

2. **What may be happening for you**
   - Structured intelligence panel with ambient glow
   - Premium card design with gradient backgrounds

3. **What they may be reacting to**
   - Dynamic signal blocks showing:
     - "How they build trust"
     - "Safety patterns"  
     - "Current stress level"
   - Each expandable on hover for detail

4. **What may be happening between you**
   - Relational context in secondary panels
   - Tension indicators in relational maps

5. **Timing / pressure context**
   - "Current timing" signals
   - "What may be heightened right now"
   - Timing timeline artifact with pressure zones

6. **What may help next**
   - Action modules (not plain buttons):
     - "What makes you say that?"
     - "Try another approach"
     - "Practice the conversation"
     - "That doesn't sound like them"

### Implementation:
- **Chat Thread:** Generative UI panels with structured blocks
- **Branch Thread:** Scenario cards with perspective shifts
- **Artifacts:** Live visualizations (Relational Map, System View, Timing View)
- **Canvas Field:** Artifact grid with real rendering, not placeholders

## 6. Relationship Overlay Logic - EXPLAINED ✅

### Where It Appears:

**Onboarding (Step 2):**
> "When another person is added, DEFRAG compares both profiles and the live situation to show where meaning may be splitting."

**Dashboard (Baseline section):**
> "Your baseline helps Defrag understand how you tend to process connection, conflict, and timing."

**Dashboard (Completion step):**
> "Add relationships to see how dynamics shift when both profiles are compared"

**Workspace Empty States:**
> "This activates when another person is added to the conversation"

**System View Artifact:**
> "Relationship overlay appears after both baselines are connected"

### Language Pattern:
- "Overlay" used intentionally (not merge, combine, or blend)
- Emphasis on comparison, not prediction
- "Where meaning may be splitting" = core value prop
- Privacy-first: "Compares synthesized patterns, not raw birth data"

## 13. GitHub Integration Notes

### Ready for AI SDK Integration:
**Structured Outputs:**
- Chat response schema matches visible UI structure
- Sources array → signal blocks
- Follow-up actions → action modules
- Artifacts → real component rendering

**Response Schema:**
```typescript
{
  acknowledgment: string
  yourExperience: string
  theirExperience: string
  dynamic: string
  timing: string
  nextSteps: string
  sources: Array<{
    name: string
    description: string
    detail: string
  }>
  followUpActions: Array<{
    label: string
    action: string
  }>
  artifacts?: Array<{
    type: 'relational-map' | 'system-view' | 'timing-view'
    data: object
  }>
}
```

**Component Architecture:**
- `<ChatThread>` → renders structured response panels
- `<CanvasField>` → renders artifacts from schema
- `<BranchThread>` → renders scenario simulations
- All TypeScript, fully typed, ready for backend
